# Requisitos Funcionais
## São as funções que o sistema deve realizar.

- Cadastro de usuários
- Login e logout
- Recuperação de senha
- Pesquisa de produtos/informações
- Emissão de relatórios
- Envio de notificações
- Cadastro, edição e exclusão de dados
- Processamento de pagamentos
- Integração com APIs externas
- Controle de permissões de acesso

# Requisitos não funcionais
## São características de qualidade e desempenho do sistema.

- Segurança dos dados
- Tempo de resposta rápido
- Interface intuitiva e fácil de usar
- Compatibilidade com diferentes dispositivos
- Disponibilidade do sistema 24h
- Escalabilidade
- Facilidade de manutenção
- Backup automático
- Confiabilidade
- Desempenho sob grande número de acessos
